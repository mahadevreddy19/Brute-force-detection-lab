# Brute Force Attack Detection using Windows Event Logs

## Objective
To detect brute-force login attempts by analyzing Windows Security Event Logs.

## Environment
- Operating System: Windows
- Tool Used: Event Viewer

## Attack Simulation
Multiple failed login attempts were intentionally generated by entering incorrect passwords multiple times.

## Logs Analyzed
- Event ID: 4625 (Failed Logon)
- Log Type: Security Logs
- Task Category: Logon

## Key Observations
- Multiple failed login attempts observed within a short time frame
- Logs marked as "Audit Failure"
- Same system and authentication context involved

## Event Details Analysis
- Logon Type: 2 (Interactive logon)
- Account Name: STARK-INDUSTRY$
- Indicates failed authentication attempts on the local system

## Detection Logic
If multiple Event ID 4625 logs occur in a short time period → possible brute-force attack

## Conclusion
The repeated failed authentication attempts clearly indicate brute-force behavior based on Windows Security logs.
