# Brute Force Attack Detection using Windows Event Logs

## Objective
To detect brute-force login attempts using real Windows Security logs.

## Environment
- OS: Windows
- Tool: Event Viewer

## Attack Simulation
Multiple failed login attempts were generated by entering incorrect passwords repeatedly.

## Logs Analyzed
- Event ID: 4625 (Failed Logon)
- Log Type: Security Logs
- Task Category: Logon

## Analysis
- Multiple failed login attempts observed within a short time frame
- All events marked as "Audit Failure"
- Same system context involved
- Indicates repeated authentication failures
- Failed login attempts occurred within seconds, indicating rapid or automated attack attempts
- No successful login (Event ID 4624) observed after failures, indicating unsuccessful brute-force attempt

## Event Details
- Logon Type: 2 (Interactive)
- Account Name: STARK-INDUSTRY$
- Failure Reason: Authentication failure
- Logon Type 2 indicates interactive login attempt from the local system

## Detection Logic
If multiple Event ID 4625 logs occur within a short time window → potential brute-force attack

## Findings
The repeated failed login attempts indicate brute-force attack behavior.

## Conclusion
The system experienced multiple authentication failures consistent with brute-force login attempts.
The repeated authentication failures within a short time frame confirm brute-force attack behavior based on Windows Security logs.
